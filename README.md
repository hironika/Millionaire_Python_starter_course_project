## 🛠 Технічні особливості та реалізація

У цьому проєкті я зосередилася на чистій архітектурі та надійних алгоритмах:

### 1. Модульність (Clean Code)
Код розділений на логічні блоки для полегшення підтримки:
- `gui.py` — графічний інтерфейс на Tkinter.
- `game_logic.py` — обробка правил, розрахунок виграшу та станів гри.
- `questions.py` — менеджер бази даних питань за рівнями складності.

### 2. Складні алгоритми підказок
- **50/50**: інтелектуальне видалення невірних варіантів за допомогою `random.sample`.
- **Допомога залу**: математична симуляція голосування з ваговим розподілом (пріоритет правильної відповіді).
- **Дзвінок другу**: логіка генерації поради з імовірністю помилки залежно від складності питання.

### 3. Сумісність (Deployment)
Реалізована кастомна функція `resource_path`, яка дозволяє додатку коректно знаходити картинки та звуки навіть після компіляції в `.exe` через PyInstaller.
